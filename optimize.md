# Как сделать сайт быстрее и удобнее

## Зачем ускорять?
- Чем быстрее грузится каталог — тем больше покупок (на каждую секунду задержки теряем 7% продаж).
- Многие сидят с телефона на даче — нужна адаптивная вёрстка.
- Поисковики (Яндекс, Google) любят быстрые сайты и ставят их выше.

## Что будем улучшать?

| Что | Как ускорим |
|-----|--------------|
| Категории товаров | Показываем первые 20 товаров, остальные подгружаем при скролле |
| Карточки товаров | Загружаем главное фото первым, остальные — позже |
| Фильтры (цена, тип, бренд) | Не перезагружаем страницу — используем быстрые запросы  |
| Поиск | Добавим подсказки при вводе, ищем только по нужным полям |
| Картинки товаров | Сжимаем в WebP, делаем несколько размеров для телефонов и ПК |
| Видеообзоры | Вставляем с YouTube (не храним на своём сервере) |
| Чат-поддержка | Грузим только когда клиент нажал «задать вопрос» |
| Корзина | Храним в браузере (localStorage) — работает без перезагрузки |

## Как будем делать

1. **Протестируем скорость** через Lighthouse, WebPageTest.
2. **Сожмём все фото товаров** — упакуем в формат WebP, размер уменьшим на 50–70%.
3. **Настроим кэширование** — чтобы браузер не скачивал одно и то же дважды.
4. **Удалим лишние скрипты** — оставим только нужные счётчики и чат.
5. **Сделаем адаптивную вёрстку** — меню превращается в бургер на телефонах.
